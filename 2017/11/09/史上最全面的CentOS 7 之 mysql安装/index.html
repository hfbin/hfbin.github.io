史上最全面的CentOS 7 之 mysql安装

发表于 2017-11-09 | 分类于 liunx | |

此教程只支持联网安装

前言

安装环境：CentOS 7 安装mysql 5.7

下载mysql源安装包

wget http://dev.mysql.com/get/mysql57-community-release-el7-8.noarch.rpm

安装mysql源

yum localinstall mysql57-community-release-el7-8.noarch.rpm

检查mysql源是否安装成功

yum repolist enabled | grep “mysql.-community.“


看到上面的内容表示安装成功！！！

修改默认安装的mysql版本

vim /etc/yum.repos.d/mysql-community.repo


将5.7源的enabled=1改成enabled=0。然后再将5.6源跟5.6源的enabled=0改成enabled=1即可。改完之后如下所示：

安装MySQL

yum install mysql-community-server

mysql服务

启动MySQL服务

systemctl start mysqld

查看MySQL的启动状态

systemctl status mysqld

开机启动

systemctl enable mysqld
systemctl daemon-reload

修改root本地登录密码以及一些错误解决

查看默认密码

grep ‘temporary password’ /var/log/mysqld.log

复制好密码等一下直接粘贴即可

进入mysql命令

mysql -uroot -p

这时候进入mysql的命令行里了

先查看一些默认有什么数据库

show databases;

不幸的是发现报一个错，错误如下图：

这句话的意识是：在执行此语句之前，必须使用ALTER USER语句重新设置您的密码

好错误说了叫我们改密码，那我们就改密码。

ALTER USER USER() IDENTIFIED BY ‘1010’;
或
ALTER USER ‘root’@’localhost’ IDENTIFIED BY ‘1010’;

抱着美好的希望，但是最后还是毁了。

好！！！ 到这我们继续看错误

错误意思是：您的密码无法满足当前的策略需求

在这解释一下：mysql5.7默认安装了密码安全检查插件（validate_password），默认密码检查策略要求密码必须包含：大小写字母、数字和特殊符号，并且长度不能少于8位。否则会提示ERROR 1819 (HY000): Your password does not satisfy the current policy requirements错误

这时候必须修改两个全局参数：

1、修改validate_password_policy参数的值

set global validate_password_policy=0;

2、validate_password_length参数默认为8，我修改为4

set global validate_password_length=4;

然后再执行修改密码操作 这时候就能成功的操作了

修改密码成功后继续测试一下

show databases;
